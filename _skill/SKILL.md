# landing-draft — черновой лендинг для холодного лида

## Контекст бизнеса
Михаил (Казань, Telegram @Miha0o0) делает сайты для малого бизнеса. Воронка: парсер 2GIS → фильтрация в Рокфеллере → черновой сайт → звонок «я уже собрал вам демо, скину ссылку в СМС» → предоплата → доработка. Черновик — главный продающий аргумент: владелец бизнеса открывает ссылку с телефона и должен захотеть купить.
Вход может быть любым: промпт из Рокфеллера, просто название лида («делаем Ехрем Хуща») или название + референсы/фотки. Промпт Рокфеллера не обязателен — все данные собирай сам и перепроверяй.

## Рабочие файлы в репозитории (читай перед работой)
- `_pipeline/leads-check.md` — проверенная база лидов с приоритетами и вердиктами (кому звонить, у кого псевдосайт, кто закрыт).
- `_pipeline/leads-status.md` — журнал статусов: кому сделан черновик, итоги звонков. Обновляй после каждого действия.
- `_skill/niche-styles.md` — библиотека стилей по нишам + референсы Михаила. Выбирай стиль отсюда; когда Михаил присылает референс — разбери его (палитра, сетка, подача) и дополни библиотеку.

## Железные правила
1. **НИ ОДНОЙ выдуманной детали.** Владелец знает свой бизнес лучше всех — одна фальшивая цифра («10 лет на рынке», «5000 довольных клиентов», выдуманное меню с ценами) убивает доверие ко всему сайту. Используй только данные из CSV/промпта + то, что нашёл в открытых источниках (VK, 2GIS, их соцсети — можно смотреть через WebFetch/WebSearch).
2. Если данных мало — лучше меньше блоков, но все честные. Вместо выдуманного меню — блок «Кухня» с общими формулировками из рубрик 2GIS.
3. **Один файл** `index.html`, полностью самодостаточный: CSS/JS инлайном, без сборки, без внешних зависимостей кроме Google Fonts и (опционально) Яндекс.Карты. Должен мгновенно открываться с телефона — владелец смотрит ссылку из СМС на ходу.
4. Сайт смотрят с телефона в первую очередь: mobile-first, крупные кнопки, click-to-call.
5. Никаких стоковых фото людей и фейковых фотографий «их» заведения. Визуал строится на типографике, цвете, градиентах, CSS-паттернах. Если у бизнеса есть VK с фото — можно честно оставить место «здесь будут ваши фотографии» со стильным плейсхолдером. Если Михаил прислал фотки заведения (вложения/зип) — сжимай (до ~200КБ каждая, webp/jpg), клади рядом с index.html и используй как реальные фото.

## Обязательная проверка лида ПЕРЕД генерацией
Каждого лида сначала проверь по свежим источникам (WebSearch по «{Название} {город}», 2GIS, новости, VK):
1. **Заведение живое?** Если закрылось (новости, «закрыто навсегда» на картах) — НЕ генерируй. Статус в базе: «ЗАКРЫТ, не звонить».
2. **Сайт уже есть?** Ищи официальный сайт, даже если в CSV колонка пустая. Проверяй ОБА картографических сервиса: 2GIS и Яндекс.Карты (найди карточку через WebSearch «{Название} {город} yandex.ru/maps/org», открой её и смотри поле сайта). Нашёл живой современный сайт — НЕ генерируй, статус «СКИП: есть хороший сайт {url}». Сайт старый/битый/только соцсети — можно генерить, но скажи Михаилу, что это скорее сценарий редизайна.
   ВАЖНО про псевдосайты: автостранички Яндекс.Бизнеса (clients.site), orgs.biz, zoon и прочие автогенерированные страницы — это НЕ сайт, генерить можно. Но если такая страница есть, обязательно предупреди Михаила: владелец может считать её «своим сайтом», подготовь контраргумент (не их домен, нет в поиске, нет меню/брони, исчезнет вместе с карточкой).
3. Заодно собери реальные факты для лендинга (кухня, фишки, средний чек, рейтинг).

## Какие данные брать из строки CSV
- Наименование, Рубрики (сфера), Описание
- Адрес + Район + Город, Широта/Долгота (для карты)
- Часы работы (переформатируй красиво: «Пн–Чт 10:00–24:00…»)
- Рейтинг + Количество отзывов (реальное соцдоказательство: «4.8 ★ · 473 отзыва на 2GIS»)
- Телефон 1 (tel:-ссылка), WhatsApp/Telegram/VK (кнопки)
- Перед генерацией, если есть VK или сайт — открой через WebFetch, вытащи реальные факты (кухня, фишки, атмосфера).

## Уровень качества (стиль Apple/Stripe/Linear)
- Сначала посмотри нишу лида в `_skill/niche-styles.md` и следуй ей; если ниши нет — придумай стиль и добавь запись в библиотеку.
- Дизайн-система: 1 акцентный цвет под сферу (ресторан — тёплые/глубокие тона; строители — уверенные тёмные + оранж; медицина — чистые холодные), нейтральная база, много воздуха.
- Типографика: Google Fonts, пара «выразительный дисплейный + чистый текстовый» (например Unbounded/Manrope, Playfair/Inter — выбирай под сферу), кегль hero ≥ 44px на десктопе.
- Плавные появления при скролле (IntersectionObserver), лёгкие hover-эффекты. Без тяжёлых библиотек.
- Тёмная или светлая тема — по сфере: рестораны/бары чаще выигрышно смотрятся в тёмной.

## Структура лендинга (адаптируй под данные)
1. **Hero**: название, одна сильная строка о сути (из рубрик/описания), рейтинг 2GIS, CTA «Позвонить» + «WhatsApp».
2. **О заведении / Почему сюда**: 3–4 карточки только из реальных фактов (рубрики, район, реальные фишки из VK).
3. **Меню/Услуги**: только если есть реальные данные; иначе аккуратный тизер без цен.
4. **Отзывы/рейтинг**: реальная цифра с 2GIS со ссылкой. Не выдумывай тексты отзывов; можно процитировать реальные из 2GIS/VK, если открывал их.
5. **Карта и контакты**: адрес, часы, телефон, мессенджеры. Карта — статичная через Яндекс static-maps по координатам или стильный блок с адресом.
6. **Футер**: «Демо-версия сайта · Разработка: Михаил» со ссылкой на Telegram Михаила: https://t.me/Miha0o0 (кликабельно, скромно, без навязчивости).
7. Плашка сверху или снизу: тонкая, ненавязчивая — «Это демо. Финальная версия делается под ваши пожелания». Она снимает претензии к неточностям и открывает разговор о доработке.
